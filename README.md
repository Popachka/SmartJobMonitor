# JobMonitor

JobMonitor - Telegram-сервис для автоматического мониторинга IT-вакансий и персонального отбора под профиль кандидата.
Проект снижает шум в вакансиях: собирает сообщения из каналов, извлекает структуру и отправляет только релевантные предложения.

**Статус проекта:** Active development, production-ready beta  
**Ссылка на бота:** [Открыть бота](https://t.me/JobMonitorIT_BOT)

## Зачем?

**Цель** JobMonitor экономит время IT-кандидата: бот сам мониторит Telegram-каналы и присылает только релевантные вакансии по резюме и фильтрам.

### Пример сценария
1. Пользователь нажимает `/start` и получает команды `/settings`, `/profile`, `/help`.
2. Пользователь открывает `/profile` и выбирает: заполнить форму через mini-app или загрузить резюме PDF.
3. Inline `web_app` кнопки открывают отдельные mini-app страницы `specialty`, `format`, `salary`.
4. JobMonitor присылает только подходящие вакансии из отслеживаемых каналов.


## Что это

JobMonitor помогает кандидатам в IT не просматривать вручную десятки Telegram-каналов.

### Для кого
- разработчики и инженеры, ищущие вакансии через Telegram;
- кандидаты, которым нужен персональный поток релевантных предложений.

### Ключевые возможности
- сбор вакансий из Telegram-каналов;
- определение, является ли сообщение вакансией;
- AI-парсинг вакансии и профиля кандидата;
- фильтрация по профилю и настройкам пользователя;
- доставка подходящих вакансий через бота.

### Ограничение текущей версии
- mini-app должен быть опубликован на HTTPS и доступен по `MINI_APP_BASE_URL`.

## Как это работает

1. **Передача профиля** - пользователь отправляет резюме (PDF).
2. **Ручная настройка** - пользователь может изменить профиль через mini-app (`/settings`).
3. **Извлечение структуры** - AI выделяет специализацию, язык, стек, опыт, зарплату и формат работы из PDF.
4. **Отбор и доставка** - базовый отбор идет по профилю, доп. фильтры применяются только при включении пользователем.

## Прозрачность отбора

- предфильтр по пересечению специализаций и основных языков;
- фильтр по опыту при включенном пороге;
- фильтр по зарплате в строгом режиме и при сопоставимой валюте;
- фильтр по формату работы (soft/strict), который пользователь настраивает в профиле.

## Архитектура и поток данных

### Диалоговый движок бота (aiogram routers + FSM)

- входящие сообщения и callback обрабатываются роутерами в `app/telegram/bot/routers`;
- FSM используется для сценария загрузки резюме (`main_menu` / `waiting_resume` / `processing_resume`);
- тексты и форматирование ответов вынесены в `app/telegram/bot/views`;
- mini-app страницы (`specialty/format/salary`) открываются по inline web_app кнопкам;
- шаблон mini-app отдается отдельным FastAPI ASGI приложением на Jinja + статике.

1. **Источники** - Telethon читает сообщения из заданных каналов.
2. **Mirror-канал** - сообщения пересылаются в служебный канал для стабильной обработки.
3. **AI-парсинг** - вакансии классифицируются и структурируются, резюме превращается в профиль поиска.
4. **Хранение и матчинг** - данные сохраняются в PostgreSQL, сервис матчинга выполняет предфильтр и доменные проверки.
5. **Доставка** - aiogram-бот отправляет релевантные вакансии пользователям.

## Mini-app

- Бот ожидает URL mini-app в переменной `MINI_APP_BASE_URL`.
- `make run` поднимает основной runtime целиком: bot, scraper и mini-app.
- `make run-miniapp` оставлен для изолированной отладки только mini-app сервера.
- Локальный сервер слушает `MINI_APP_SERVER_HOST:MINI_APP_SERVER_PORT`.
- Mini-app шаблоны и статика находятся в `app/telegram/miniapp/`.
- Публичные страницы: `GET /miniapp/specialty`, `GET /miniapp/format`, `GET /miniapp/salary`.
- Для Telegram mini-app нужен HTTPS URL, поэтому локально пробрасывайте туннель на `MINI_APP_SERVER_PORT` и указывайте его в `MINI_APP_BASE_URL`.
- В этой версии формы демонстрационные: сохранение и `POST /api/miniapp/save-section` не подключены.
